# bbsDemo 前端接口文档

## 项目概述

bbsDemo 是一个基于 Vue 3 + TypeScript + Element Plus 的论坛前端项目，实现了用户认证、帖子管理、评论、点赞、收藏等核心功能。

## 技术栈

- **框架**: Vue 3.5.29 + TypeScript 5.9.3
- **UI 组件库**: Element Plus 2.13.3 (按需引入)
- **路由**: Vue Router 5.0.3
- **状态管理**: Pinia 3.0.4
- **HTTP 客户端**: Axios 1.13.6
- **构建工具**: Vite 7.3.1
- **代码检查**: vue-tsc 3.2.5
- **自动导入**: unplugin-auto-import 21.0.0, unplugin-vue-components 31.0.0
- **图片处理**: 原生 Canvas API (替代 vue-cropper 1.1.4)
- **开发工具**: Vite Plugin Vue DevTools 8.0.6

## API 服务

### 基础配置

API 服务基于 Axios 封装，配置文件位于 `src/services/api.ts`：

- **基础 URL**: `http://localhost:8080`
- **超时时间**: 10000ms
- **Content-Type**: `application/json`

### 请求拦截器

自动在请求头中添加 Token：

```typescript
config.headers.Authorization = `Bearer ${token}`
```

### 响应拦截器

- 自动保存后端返回的 token
- 处理 401 未授权错误，自动跳转到登录页

## 错误处理

所有 API 请求都会返回 Promise，建议使用 try-catch 进行错误处理：

```typescript
try {
  const response = await postApi.getPosts({ page: 1, page_size: 10 })
  // 处理成功响应
} catch (error: any) {
  // 处理错误
  const errorMessage = error.response?.data?.error || '请求失败'
  console.error(errorMessage)
}
```

## 响应格式

标准响应格式：

```typescript
{
  "code": 200,        // 状态码
  "message": "success", // 消息
  "data": { ... },    // 数据
  "total": 100        // 总数（分页时）
```

所有列表接口都支持分页：

- `page`: 当前页码（从 1 开始）
- `page_size`: 每页数量

返回数据包含 `total` 字段表示总数。

## 注意事项

1. 所有需要认证的接口会自动携带 Token
2. Token 过期会自动跳转到登录页
3. 文件上传需要设置 `Content-Type: multipart/form-data`
4. 建议使用 async/await 处理异步请求
5. **ID 类型变更**：所有 ID（包括用户 ID、帖子 ID、评论 ID 等）现在使用字符串类型传输，以支持大整数 ID

## 浏览量更新机制

### 功能说明

帖子详情页实现了浏览量自动更新功能：

- ✅ 后端在获取帖子详情时自动增加浏览量
- ✅ 前端延迟重新获取帖子详情以确保显示最新浏览量
- ✅ 支持多种浏览量字段名（`views`、`view_count`、`visits`、`visit_count`）
- ✅ 详细的日志输出便于调试

### 技术实现

由于后端在获取帖子详情时增加浏览量，前端需要延迟重新获取以确保显示最新数据：

```typescript
// 获取帖子详情
const fetchPostDetail = async () => {
  try {
    isLoading.value = true
    const response = await postApi.getPostDetail(postId.value)
    post.value = response.post || response
    
    // 延迟重新获取帖子详情，确保获取到最新的浏览量
    setTimeout(async () => {
      try {
        const refreshedResponse = await postApi.getPostDetail(postId.value)
        post.value = refreshedResponse.post || refreshedResponse
      } catch (err) {
        console.error('重新获取帖子详情失败:', err)
      }
    }, 1000) // 延迟1秒后重新获取
  } catch (err: any) {
    console.error('获取帖子详情错误:', err)
  } finally {
    isLoading.value = false
  }
}

// 获取浏览量（支持多种字段名）
const getPostViews = () => {
  if (!post.value) return 0
  
  // 尝试多种可能的字段名
  return post.value.views || 
         post.value.view_count || 
         post.value.visits || 
         post.value.visit_count || 0
}
```

### 使用方式

1. 用户访问帖子详情页
2. 后端自动增加浏览量
3. 前端延迟1秒后重新获取帖子详情
4. 显示更新后的浏览量

5. 取消拉黑后，用户状态标记为"已取消"

### 技术实现

黑名单管理通过 `unblocked_at` 字段实现，取消拉黑后保留记录：

```typescript
// 检查用户是否已被取消拉黑
const isUserUnblocked = (user: any) => {
  return user.unblocked_at !== null
}

// 获取用户状态文本
const getUserStatusText = (user: any) => {
  if (isUserUnblocked(user)) {
    return '已取消'
  }
  return '已拉黑'
}

// 获取拉黑时间或取消时间
const getBlockTime = (user: any) => {
  if (isUserUnblocked(user)) {
    return formatDate(user.unblocked_at)
  }
  return formatDate(user.blocked_at)
}
```

**字段说明**:
- `blocked_at`: 拉黑时间
- `unblocked_at`: 取消拉黑时间（null 表示未取消拉黑）
- 可以通过 `unblocked_at` 字段判断用户是否已取消拉黑

## 头像裁剪功能

### 功能说明

用户个人中心支持头像裁剪功能，使用原生 Canvas 实现：

- ✅ 支持用户自由框选头像区域
- ✅ 1:1 比例裁剪
- ✅ 可拖动和调整裁剪框
- ✅ 实时预览裁剪效果
- ✅ 自动将裁剪后的图片上传

### 使用方式

1. 点击 "更换头像" 按钮
2. 选择图片文件
3. 在弹出的裁剪对话框中调整裁剪区域
4. 点击 "确认" 按钮完成上传
5. 或点击 "取消" 按钮放弃上传

### 技术实现

使用原生 Canvas API 实现，主要功能包括：

1. **图片加载与缩放**：自动将图片缩放到合适大小并居中显示
2. **裁剪框操作**：
   - 支持鼠标拖动移动裁剪框
   - 支持拖拽四角调整裁剪框大小
   - 显示半透明遮罩和蓝色边框
3. **裁剪逻辑**：
   - 使用临时 Canvas 进行精确裁剪
   - 转换为 JPEG 格式 Blob 对象
   - 通过 `uploadAvatar` 接口上传

### 核心代码

```typescript
// 初始化裁剪器
const initCropper = () => {
  if (!cropperCanvas.value) return
  
  const canvas = cropperCanvas.value
  const ctx = canvas.getContext('2d')
  if (!ctx) return
  
  // 清空画布
  ctx.clearRect(0, 0, canvas.width, canvas.height)
  
  // 加载图片
  const img = new Image()
  img.onload = () => {
    // 计算缩放比例
    const maxSize = Math.min(canvas.width, canvas.height) * 0.8
    const scale = Math.min(maxSize / img.width, maxSize / img.height)
    const scaledWidth = img.width * scale
    const scaledHeight = img.height * scale
    
    // 居中显示
    const x = (canvas.width - scaledWidth) / 2
    const y = (canvas.height - scaledHeight) / 2
    
    cropperPosition.value = { x, y }
    cropperScale.value = scale
    
    // 绘制图片
    ctx.drawImage(img, x, y, scaledWidth, scaledHeight)
    
    // 绘制裁剪框
    drawCropBox(ctx)
  }
  img.src = cropperImage.value
}

// 确认裁剪
const confirmCrop = async () => {
  const canvas = cropperCanvas.value
  if (!canvas) return
  
  // 创建临时 canvas 用于裁剪
  const tempCanvas = document.createElement('canvas')
  tempCanvas.width = cropBox.value.width
  tempCanvas.height = cropBox.value.height
  
  const ctx = tempCanvas.getContext('2d')
  if (!ctx) return
  
  // 加载原图
  const img = new Image()
  img.onload = async () => {
    // 计算实际裁剪位置和大小
    const { x, y, width, height } = cropBox.value
    const { x: px, y: py } = cropperPosition.value
    
    // 绘制裁剪区域
    ctx.drawImage(
      img,
      (x - px) / cropperScale.value,
      (y - py) / cropperScale.value,
      width / cropperScale.value,
      height / cropperScale.value,
      0, 0, width, height
    )
    
    // 转换为 Blob
    tempCanvas.toBlob(async (blob) => {
      if (blob) {
        await userStore.uploadUserAvatar(blob)
        showCropper.value = false
      }
    }, 'image/jpeg', 0.9)
  }
  img.src = cropperImage.value
}
```
